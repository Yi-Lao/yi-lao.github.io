在GitHub上搭建博客

发表于 2017-08-11

Node.js+Hexo+Github

Node.js

  • NodeJS是一个使用了Google高性能V8 引擎 的服务器端JavaScript实现。它提供了一个（几乎）完全非阻塞I/O栈，与JavaScript提供的闭包和匿名函数相结合，使之成为编写高吞吐 量网络服务程序的优秀平台。-
    Hexo
  • 一个基于 Nodejs 的静态博客网站生成器-
  • 支持 Markdown
  • 仅需一道指令即可部署到 GitHub Pages 和 Heroku
  • 已移植 Octopress 插件
  • 快效率、高扩展性、自订性
  • 兼容于 Windows, Mac & Linux

Git

  • 一个免费、开源的分布式版本控制系统，可以有效、高速的处理从很小到非常大的项目版本管理-

安装配置Node.js、Hexo及Github

安装Node.js

  • 下载地址

安装Git

  • 下载地址

安装Hexo
– 安装完成以上程序，打开Git-bash，执行

npm install -g hexo-cli

创建GitHub的Repository

创建GitHub账号

  • 注册登录

创建与账号同名的Repository

  • 一定要同名的Repository
  • e.g：Github账号为yourname，则Repository名称为yourname.github.io

使用Hexo进行本地建站

  • 选择一个本地的文件夹(e.g:D:/hexo)，并安装npm，执行-

    $hexo init D:/hexo
    $cd D:/hexo
    $npm install

  • 启动本地部署好的Hexo-

$hexo server

  • 访问个人博客网址http://localhost:4000/，出现主页即为成功-
  • 此时D:/hexo文件夹下文件目录为-

    .
    ├── _config.yml // 网站的配置信息，你可以在此配置大部分的参数。
    ├── package.json
    ├── scaffolds // 模板文件夹。当你新建文章时，Hexo会根据scaffold来建立文件。
    ├── source // 存放用户资源的地方
    │ 　　├── _drafts
    │ 　　└── _posts
    └── themes // 存放网站的主题。Hexo会根据主题来生成静态页面。

配置SSH

生成ssh

  • 生成SSH，执行-

    $ssh-keygen -t rsa -C “your e-mail” //your e-mail为注册Github时的邮箱

  • 然后一直enter，生成文件在C:/Users/user/.ssh/id_rsa.pub-
    复制ssh key到Github账户设置中
    – 打开~/.ssh/id_rsa.pub文件，全选复制
    – 进入Github->setting->SSH and GPG keys->New SSH key,填写Title，讲复制的ssh key复制到Key框中，点击Add key。
    测试ssh配置

  • 执行:-

    $ssh -T git@github.com

如果返回：

Hi Yi-Lao! You’ve successfully authenticated, but GitHub does not provide shell access.

本地博客同步到Github

配置Github账户

$git config –global user.name “username”
$git config –global user.email “email”

  • 查看是否配置成功-

    $git config //查看已设配置

网站发布到Github的repository中

  • 修改D:/hexo目录下_config.yml-

    Deployment

    Docs: http://hexo.io/docs/deployment.html

    deploy:
    　type: git
    　 repo: git@github.com:MyGithub/MyGithub.github.io

  • (1) 其中MyGithub替换成你的Github账户名; (2)注意在yml文件中，:后面都是要带空格的。
  • 保存后访问http://MyGithub.github.io，出现主页即为成功。-
